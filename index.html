业务概述

该模型主要用于企业级云成本管理与优化（FinOps）。它通过整合多源数据（实际消费、预算预测、预留实例、商务承诺等），主要解决了以下核心业务问题：

01
Cost Allocation
云精细化成本分摊与"谁消费、谁买单"的问责机制。通过多层级组织架构映射与多维度标签分析，将云消费精准归属到业务单元，实现内部财务转资与成本问责。
02
Visibility & Tracking
成本全生命周期监控与多视角财务度量。提供账面与实际净成本对账、商务节省可量化，消除财务与运维部门的跨部门沟通壁垒。
03
Performance
多粒度性能优化与大数据量下的秒级报表响应。采用多层事实表设计（Layered Fact Grain），高层看聚合看板快如闪电，下层钻取明细实时精准。
04
Forecast & Commitment
动态趋势预测与承诺指标达成监控。实时展示实际发生额与预测对比的全景视图，监控 MACC 消耗完成进度，防止承诺达标失败。
05
Reservation Optimization
RI/SP 优化与降本增效诊断。监控预留资源利用率，量化"未实现节省"指标，辅助 FinOps 团队进行精准的资源购买决策。
06
Anomaly Detection
异常消费行为的智能识别与预警。交叉比对运行时间与非工作日、非生产环境，自动高亮异常资源，帮助运维团队快速止血。

业务概述

该模型是一个专门用于企业级云成本节约追踪与 FinOps（云财务运营）效能分析的 Power BI 语义模型。它通过整合多源数据（包括 Azure 账单、SharePoint 目标配置、Azure 顾问 API 等），主要解决了以下六大核心业务问题：

01
Accountability & BU Slicing
跨团队/业务单元的"云节约贡献"透明化与责任归属。通过 dim_BU 维度表将节约数据与 27 个业务单元绑定，并内置 28 个 RLS 角色，各 BU 只能查看本部门数据。
02
Cost Savings Tracking
多渠道降本倡议的统一追踪与成效量化。将预留实例节省、降配优化节省、关键资源重选与架构重构优化四种手段整合到统一的真理源中。
03
Target Achievement
年度降本目标的达成率监控与差距分析。通过 DAX 度量值动态计算目标完成百分比，配合 Gauge 可视化直观展示各 BU 达标进度。
04
Trend Forecasting
财年终点预测与 Actual + Forecast 混合趋势规划。自动将已实现节约与未来预测融合，生成 FY Projection，提前预知全年 FinOps 目标达成情况。
05
Opportunity Prioritization
错失机会的量化与优化路线图构建。对接 Azure Advisor 推荐 API 量化未采纳降本潜力，计算每月潜在损失，帮助 IT 团队合理排列优化优先级。
06
Data Silos Reduction
消除组织内部的财务对账信息孤岛。基于 Databricks 铜-银-金三层架构进行 ETL 清洗与标准化，自动化月度刷新，实现全公司云节约效益的"一盘棋"管理。

业务概述

该模型主要用于 AIA Group 的云服务与 IT 基础架构托管（Kyndryl）日常消费监控、预算管控与高管财务决策支持（FinOps）。通过整合 Azure Databricks 数据仓库中不同层级的数据，主要解决了以下五个方面的核心业务问题：

01
Data Silos & Transparency
跨系统、多服务形态的云与托管成本数据孤岛。首次打通公有云消费与 Kyndryl DCM、RFS、CMS 等多种传统 IT 托管支出，建立统一真理源，多币种自动转化为 USD 本位币。
02
Budget Management
Actual vs. Forecast vs. Target 三位一体实时预算对账。通过 Variance 偏差度量值动态评估各团队预测精度，防止资金闲置或超支，实现年中动态干预。
03
Anomaly Detection
多视角云成本异常检测与"触发器"预警机制。基于 SharePoint 商务基准线，通过 Trigger 指标自动监控消费偏差，达到 50% 警戒线时自动标红警示。
04
Intelligent Time Intelligence
智能时间序列与实际/预测期平滑切换。自动识别数据断点，派生出 YearLabel（2026A 实际 / 2026F 预测），保证全财年滚动预测在时间维度上的严谨性。
05
Data Governance & RLS
多租户环境下敏感财务数据的行级安全隔离。建立包含 27 个业务单元在内的 28 个 RLS 矩阵，普通经理限本部门，高管与审计拥有全透视权限。

Cloud Cost Management Report

Cloud Cost Management Report — 管理云成本优化全流程，跟踪预算执行、资源利用率及成本分配，推动 FinOps 持续改进。
正在加载报告

业务概述

该模型是一个专门为企业级云财务运营（FinOps）设计的 Azure 云成本管理分析平台。通过整合来自 Azure Cost Management API 的海量账单数据，并采用 DirectQuery + Import 混合存储模式的星型架构，主要解决了以下核心业务问题：

01
Cost Transparency & Allocation
云资源消耗的"黑盒"状态与全维度成本透明化。通过标签穿透分析成本中心、WBS 编码、业务应用系统及环境属性，为跨部门内部转资和精准摊销提供数据根基。
02
Savings Analysis
商务折扣与 RI/SP 承诺项目的效益量化与节约追踪。提供目录原价、合同协议价、净有效成本多视角对账，量化折扣节省、承诺节省与合同谈判节省的 ROI。
03
Budgeting & Forecasting
向"事前预防"转变的滚动预测与预算管控。将实际成本与基于 12 个月历史趋势的预测值平滑拼接，生成全年滚算总花费预测，月中即可洞察超支风险。
04
Performance vs. Real-time
海量账单明细下的秒级宏观汇总与实时微观钻取。明细事实表采用 DirectQuery 确保资源级实时查询，维度与预测表采用 Import 实现高管看板亚秒级响应。
05
Resource Optimization
资源运维视角的闲置浪费与异常负载识别。结合 Hrs、CPU 核心小时数等指标，对非生产环境在周末异常高耗能行为自动染色标红，辅助缩容降配决策。
06
Data Governance via RLS
多租户 BU 财务敏感数据的行级安全隔离。配置 28 个 RLS 角色（1 个 Full Access + 27 个地区/BU 专属），安全过滤自动下推至 Databricks 执行。
